## 📄 页面清单

| 页面名称 | 类型 | 优先级 | 说明 |

#### 功能说明

| 区域 | 功能 | 交互 |
|------|------|------|
| **顶部栏** | 标题 + 设置入口 | 点击设置进入设置页 |
| **账本卡片** | 展示每个账本的概览信息 | 点击卡片进入账本详情 |

- ✅ 卡片式设计，视觉层次清晰
- ✅ 横向滚动账本列表（如果账本多）
- ✅ 数字突出，重要信息居中
- ✅ Material Design 阴影和间距
- ✅ 配色：主色调（蓝色/紫色）+ 强调色（绿色收入/红色支出）

```
├─────────────────────────────────┤

│  📂 分类选择                     │
│  ┌─────────────────────────────┐ │

│  │  [选择分类 ▼]               │

│  └─────────────────────────────┘ │
│                                 │
│  📅 日期选择                     │
│  ┌─────────────────────────────┐ │

│  └─────────────────────────────┘ │
│                                 │
│  📝 标签/备注                    │
│  ┌─────────────────────────────┐ │
│  │  买菜                       │

│  └─────────────────────────────┘ │
│                                 │
│  👛 账本选择                     │
│  ┌─────────────────────────────┐ │
│  │  日常账本        ▼            │  ← 下拉选择
│  └─────────────────────────────┘ │
│                                 │
│  [  删除 ]  [ 保存 ]            │  ← 操作按钮
│                                 │
└─────────────────────────────────┘
```

#### 功能说明

| 区域 | 功能 | 验证 |
|------|------|------|
| **金额输入** | 输入交易金额 | 必填，正数，最多两位小数 |
| **收支类型** | 选择收入或支出 | 单选，默认支出 |
| **分类选择** | 选择消费分类 | 必填，支持快速新增 |
| **日期选择** | 选择交易日期 | 必填，默认今日 |
| **标签备注** | 添加备注信息 | 可选，便于回忆 |
| **账本选择** | 选择所属账本 | 必填，默认第一个账本 |

#### 交互流程

```
1. 首页 → 底部"记录"或快速按钮
   ↓
2. 进入记录列表/新增记录
   ↓
3. 点击"+"或"新增"
   ↓
4. 打开记录表单（模态框/新页面）
   ↓
5. 填写信息
   ↓
6. 点击"保存"
   ↓
7. 返回列表，刷新显示新记录
```

#### 设计要点

- ✅ 表单字段明确，分组展示
- ✅ 验证错误实时提示（在字段下方红色文字）
- ✅ 保存前检查必填字段
- ✅ 输入框有明确的焦点状态（Material Design）
- ✅ 支持快速分类选择（标签云）

## 🎬 交互设计

### 常见交互场景

#### 场景 1: 新增记录流程

```
1. 用户在首页看到"+ 新增记录"或"快速记账"按钮
2. 点击进入记录表单页面
3. 逐步填写：
   - 金额（必填，实时验证）
   - 收/支类型（默认支出）
   - 分类（可快速新增）
   - 日期（日期选择器）
   - 备注（可选）
   - 账本（下拉选择）
4. 点击"保存"按钮
5. 返回列表，显示 Toast "记录已保存"
6. 列表自动刷新，新记录出现在顶部
```

#### 场景 2: 编辑记录流程

```
1. 用户在账本详情列表中看到记录
2. 点击记录卡片进入详情页（或滑动显示编辑按钮）
3. 进入编辑模式（表单预填充当前数据）
4. 修改相关字段
5. 点击"保存"
6. 返回列表并刷新
```

#### 场景 3: 删除记录流程

```
1. 用户在列表中长按记录 或 点击删除按钮
2. 弹出确认对话框
3. 用户确认删除
4. 记录从列表中移除
5. 显示 Toast "记录已删除"
6. 如果用户点击撤销（Toast 上的按钮），恢复记录
```

#### 场景 4: 查看统计分析

```
1. 用户点击底部导航"分析"
2. 进入统计分析页面
3. 默认显示当前月份的数据
4. 用户可点击月份切换器查看其他月份
5. 图表实时更新
6. 用户可点击分类查看详细信息
```

### 微交互设计

| 交互 | 效果 | 场景 |
|------|------|------|
| **输入框焦点** | 下划线变色，弹出键盘 | 点击输入框 |
| **按钮按下** | 背景色变深/涟漪效果 | 点击任何按钮 |
| **加载动画** | 转圈菊花加载器 | 数据加载中 |
| **成功提示** | 绿色 Toast + 对勾图标 | 操作成功 |
| **错误提示** | 红色 Toast + 错误信息 | 操作失败 |
| **确认对话框** | 半透明背景 + 动画显示 | 危险操作 |
| **列表刷新** | 下拉刷新动画 | 用户主动刷新 |
| **页面转场** | 渐变/滑动动画 | 页面跳转 |
